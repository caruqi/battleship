# Морской бой

Сетевая игра «Морской бой» на двух игроков. Сервер (Express + `ws`) —
единственный источник истины: хранит расстановку кораблей каждого игрока
в памяти процесса и никогда не отправляет чужую расстановку клиенту —
только результаты выстрелов.

## Запуск локально

```bash
npm install
npm start
```

Откройте `http://localhost:3000` в двух вкладках (или на двух устройствах
в одной сети) — одна создаёт комнату, вторая вводит код.

Порт можно переопределить переменной окружения `PORT`.

## Структура

```
battleship/
├─ server/
│  ├─ index.js       # Express + WS сервер, статика фронтенда
│  ├─ rooms.js        # логика комнат, состояние партий, ходы
│  └─ shipRules.js    # валидация расстановки, флот, проверка победы
├─ public/
│  ├─ index.html
│  ├─ styles.css
│  └─ game.js          # клиентская логика, рендер, WS-клиент
└─ package.json
```

## Деплой

Нужен хостинг с постоянным Node-процессом и поддержкой WebSocket
(Render, Railway, Fly.io — подходят; Vercel serverless — нет).

Пример для **Render**:
1. Запушить репозиторий на GitHub.
2. New → Web Service → выбрать репозиторий.
3. Build Command: `npm install`
4. Start Command: `npm start`
5. Render сам пробрасывает переменную `PORT` — сервер уже её читает.

После деплоя партии не переживают перезапуск сервера (комнаты — in-memory),
это соответствует спецификации.
